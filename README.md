# Отчет по задачам для Bitrix24

Это приложение-отчет, созданное на Vue.js и Vuetify для платформы Bitrix24. Оно позволяет анализировать затраченное время по задачам.

## Возможности

- Просмотр задач в двух режимах: детально по каждой задаче или сгруппировано по сотрудникам.
- Фильтрация отчета по сотруднику, статусу задачи и дате ее завершения.
- Автоматический подсчет итоговых сумм по запланированному/затраченному времени и количеству задач.

## Технологии

- **Frontend:** Vue 3, Vuetify 3
- **Сборка:** Gulp, Webpack
- **Окружение:** Node.js, Docker

## Как запустить проект

### Способ 1: Локальная разработка (без Docker)

Этот способ подходит для быстрой разработки и внесения изменений.

1.  **Установите Node.js:** Убедитесь, что у вас установлена версия 20.x или выше.
2.  **Установите зависимости:** В корневой папке проекта выполните команду:
    ```sh
    npm install
    ```
3.  **Запустите сервер для разработки:**

    ```sh
    npx gulp serve --gulpfile gulp/gulpfile.js
    ```

    Приложение будет доступно по адресу `http://localhost:3000` и `https://localhost:3001`. Сервер будет автоматически пересобирать проект при внесении изменений в исходный код.

4.  **Сборка для продакшена:** Для создания готовой к развертыванию версии выполните:
    ```sh
    npx gulp build --gulpfile gulp/gulpfile.js
    ```
    Файлы сборки появятся в папке `dist`.

### Способ 2: Запуск через Docker

Этот способ гарантирует, что приложение будет работать в изолированном и предсказуемом окружении. Идеально подходит для развертывания или для того, чтобы избежать установки зависимостей на локальную машину.

1.  **Установите Docker:** Убедитесь, что Docker Desktop запущен.
2.  **Соберите Docker-образ:** В корневой папке проекта выполните команду. Имя образа будет `list-employees-company`.
    ```sh
    docker build -t list-employees-company .
    ```
3.  **Запустите контейнер:**

    ```sh
    docker run -d -p 3000:3000 -p 3001:3001 --name list-employees-company list-employees-company
    ```

    - `-d` - запуск в фоновом режиме.
    - `-p` - проброс портов из контейнера на ваш компьютер.
    - `--name` - удобное имя для контейнера.

4.  **Откройте приложение:** Приложение будет доступно по адресу `http://localhost:3000` и `https://localhost:3001`.

### Управление Docker-контейнером

- **Посмотреть логи (журнал) контейнера:**
  ```sh
  docker logs list-employees-company
  ```
- **Остановить контейнер:**
  ```sh
  docker stop list-employees-company
  ```
- **Запустить остановленный контейнер:**
  ```sh
  docker start list-employees-company
  ```
- **Удалить контейнер (перед этим его нужно остановить):**
  ```sh
  docker rm list-employees-company
  ```

### Сборка проекта внутри контейнера

Если вам нужно вручную запустить только сборку проекта (без сервера разработки), используйте команду `docker exec`.

- **Запустить задачу сборки `build`:**
  ```sh
  docker exec list-employees-company npx gulp build --gulpfile gulp/gulpfile.js
  ```
- **Скопировать папку `dist` из контейнера на ваш ПК:**
  ```sh
  docker cp list-employees-company:/app/dist ./
  ```
